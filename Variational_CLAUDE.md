## 2. 직무 소개

Variational이라는 **Perpetual DEX의 한국 시장 단독 담당자**.

### 핵심 목표
> 한국 시장에서 Variational의 **사용자 온보딩 극대화** + **거래량 증대**

### 세부 직무
- Variational 플랫폼에 대한 깊은 이해
- 국내 시장 반응 체크 + 피드백을 본사에 보고
- **커뮤니티 반응 보고 및 피드백** — 각 채널의 사용자 반응·의견을 수집하여 본사에 체계적으로 보고
- Variational 홍보 및 각종 SNS/홍보 채널 세팅·운영
- **마케팅 업무** — 한국 시장 대상 마케팅 전략 수립 및 실행 담당 예정
- 사용자 온보딩 + 거래량 극대화
- 혼자 한국 시장 전체를 담당 → **AI 에이전트 간 소통·브레인스토밍 필수**

## 3. AI 사용 목적

모든 업무 자동화 및 각 AI 에이전트 간 **효율적인 커뮤니케이션 + 상호 자동 업무 시스템 확립**

## 4. AI 에이전트 역할 분담 및 소통 체계

### Market Follower Agent (Internal) — Variational 데이터 수집

**역할:** Variational 자체 시장 심리·반응·질문 수집

| 수집 대상 | 세부 내용 |
|-----------|-----------|
| **Variational 반응** | X, Discord, Telegram 등 커뮤니티에서 한국 이용자들의 반응·시장 심리 취합 |
| **KPI 데이터** | Variational의 조회수, 댓글 수, 언급량 등 수치화 가능한 지표 정리 |
| **커뮤니티 질문** | 각 채널에서 올라온 질문 취합 → Q&A Agent에게 전달 |

> **긴급 알림:** Variational 관련 FUD 확산, 플랫폼 장애 불만 폭주 등 **긴급 이슈 감지 시 즉시 피넛버터 텔레그램으로 알림**

---

### Market Follower Agent (External) — 경쟁사 데이터 수집

**역할:** 경쟁사 동태 파악 + 시장 반응 수집

| 수집 대상 | 세부 내용 |
|-----------|-----------|
| **경쟁 업체 정의** | DefiLlama, CoinMarketCap 기준 **주간 거래량 상위 7개 Perpetual DEX** |
| **경쟁 업체 모니터링** | 시장 반응·심리, 각종 업데이트 소식, 동태 파악 |
| **경쟁사 KPI** | 경쟁사 거래량, SNS 반응 수치 등 비교 지표 수집 |

> **긴급 알림:** 경쟁사 대형 이벤트·에어드롭 발표, 주요 업데이트 등 **긴급 이슈 감지 시 즉시 피넛버터 텔레그램으로 알림**

---

### Report Agent — 리포트 작성

**역할:** Market Follower Agent (Internal + External)의 데이터를 **일간/주간 보고서**로 정리 + Trend Research Agent의 트렌드 데이터도 참고

| 보고서 항목 | 내용 |
|-------------|------|
| 거래량 추이 | 일간/주간 거래량 변화 (Variational + 경쟁사 비교) |
| 이용자 수 | 일간/주간 이용자 수 변화 추이 |
| SNS 활성도 | 언급 수, 댓글 반응 수치 |
| 커뮤니티 활성도 | 각 채널별 활성 지표 |
| 사용자 피드백 | 심리, 반응, 플랫폼에 바라는 점 |
| 경쟁사 동향 | 주요 업데이트, 이벤트, 시장 반응 비교 |
| 인사이트 | 피드백 기반 개선점 도출 |

> **보고 시간:** 한국 시간 기준 **오전 11:00** · **보고 방식:** 이메일

---

### Report Reviewer Agent — 리포트 검토

**역할:** Report Agent 리포트의 **품질 관리**

- 오류 검토 + 수정
- 추가/보완할 내용 확인
- 최종 결론 및 인사이트에 대해 **브레인스토밍**
- 최적의 전략 구상 → 리포트 퀄리티 극대화

> **최종 보고:** 피넛버터 **텔레그램** · 에이전트 간 소통 요약 포함

---

### Trend Research Agent — 트렌드 리서치

**역할:** 크립토 트렌드·밈 모니터링 + 콘텐츠 소재 발굴 + Market Follower Agent (External) 경쟁사 동태 참고

- KOL의 텔레그램방, X에서 **자주 언급되는 트렌드/밈** 리포팅
- 조회 수·댓글 수가 많은 콘텐츠 포착
- **콘텐츠 재료로 활용할 만한 소재** 리서치

---

### Contents Creator Agent — 콘텐츠 기획

**역할:** Trend Research Agent의 트렌드 소재 → **콘텐츠 아이디어 + 초안 제작**

- 콘텐츠 제작 아이디어 보고
- 예시 및 초안 직접 제작 후 보고

> **최종 보고:** 피넛버터 **텔레그램** · 에이전트 간 소통 요약 포함

---

### Q&A Agent — 커뮤니티 응대

**역할:** 커뮤니티 질문·피드백 **최적의 답변 구상**

- Market Follower Agent (Internal)이 취합한 커뮤니티 질문/피드백 수신
- 최적의 답변 방향 고민 + 답변 초안 작성

> **보고:** 피넛버터 **텔레그램 다이렉트**

---

### Performance Agent (Daily) — 단기 성과 추적

**역할:** 콘텐츠·이벤트·캠페인의 **건별 즉시 성과 측정**

| 추적 항목 | 세부 내용 |
|-----------|-----------|
| **콘텐츠 성과** | 게시 후 조회수, 좋아요, 리트윗, 댓글 수 변화 |
| **이벤트 성과** | 이벤트 참여율, 신규 유입 수, 반응 |
| **캠페인 성과** | 프로모션별 클릭율, 참여율, 전환율 |

> **보고:** 실행 후 **24~48시간 내** · 피넛버터 **텔레그램 다이렉트**

---

### Performance Agent (Weekly) — 장기 성과 분석

**역할:** 전체 전략의 **ROI·성장 추이 분석** + POC Agent에게 데이터 제공

| 분석 항목 | 세부 내용 |
|-----------|-----------|
| **거래량 추이** | 주간/월간 거래량 변화 + 전략 실행과의 상관관계 분석 |
| **온보딩 전환율** | 유입 → 가입 → 첫 거래까지의 전환 흐름 분석 |
| **채널별 ROI** | 어떤 채널/캠페인이 가장 효과적이었는지 비교 |
| **성장 트렌드** | 월간 성장률, 경쟁사 대비 포지션 변화 |

> **보고:** **주간 1회** · 피넛버터 **텔레그램 다이렉트** + POC Agent에 데이터 전달

---

### Promotion Agent — 마케팅 기획

**역할:** 한국 시장 대상 **마케팅 전략 수립 + 캠페인 기획**

| 업무 항목 | 세부 내용 |
|-----------|-----------|
| **이벤트 기획** | 사이드 이벤트, 에어드롭, 트레이딩 대회 등 기획안 작성 |
| **KOL 협업** | 한국 크립토 KOL 리스트업 + 협업 제안서 초안 작성 |
| **프로모션 기획** | 신규 유저 온보딩 프로모션, 거래량 부스트 캠페인 설계 |
| **채널 전략** | SNS 채널별 최적 콘텐츠 포맷·게시 타이밍 전략 |
| **예산 제안** | 캠페인별 예상 비용 + 기대 효과 정리 |

> **보고:** 피넛버터 **텔레그램 다이렉트** · 참고: Market Follower (External) + Trend Research Agent + Performance Agent 데이터

---

### POC Agent — 상사 보고서 작성

**역할:** 피넛버터에게 보고된 **모든 텔레그램 메시지를 총 정리**하여 상사에게 **최종 리포트 작성**

#### 리포트 구성

```
1. Executive Summary — 핵심 성과 3줄 요약 + 의사결정 필요 사항
2. 핵심 지표 대시보드 — 거래량/이용자/SNS (전주 대비 ↑↓%) + 경쟁사 비교
3. 시장 심리 & 커뮤니티 동향
4. 콘텐츠 & 트렌드
5. 마케팅 활동 & 성과
6. 전략 성과 분석
7. 커뮤니티 피드백 요약
8. 제안 & 다음 스텝
```

> **원칙:** 구조화 · 시각화 · 간결성 · 근거 기반 · 액션 지향
> **보고:** 주간 1회 (금요일) · 피넛버터 확인 후 → **상사에게 이메일 발송** (PDF/문서)

|------|--------------|------|

| 📈 **성과 + 종합 보고 채널** | Performance Agent (Daily/Weekly) + POC Agent | OFF |

> 긴급 알림 채널만 알림 ON, 나머지는 묵음 + 필요 시 확인

## 7. 보안 정책

- 모든 API 키·봇 토큰은 **`.env` 파일**에 저장, 코드에 직접 기입 금지
- `.env`는 **`.gitignore`**에 등록하여 Git 업로드 차단
- 봇 토큰에는 **최소 권한만** 부여 (메시지 전송만 가능 등)
- 토큰은 **월 1회 주기적 교체** 권장
- `.env.example`에는 플레이스홀더만 기입 (실제 값 절대 금지)
